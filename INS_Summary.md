# 智能车惯性导航系统（INS）技术总结文档

> **项目名称**: new_ins — 基于三层融合架构的二维平面惯性导航系统  
> **硬件平台**: Infineon TC377 (Aurix TriCore) + 逐飞开源库  
> **作者**: Daydreamer / LateRain

## 一、系统总览

本系统为智能车竞赛设计的 **二维平面惯性导航系统**，采用 **"六轴卡尔曼 → 磁力计校正 → 航位推算"** 三层融合架构，实现了从传感器原始数据到车辆位置姿态的完整解算链路。系统同时集成了 **轨迹录制与Pure Pursuit循迹** 功能，支持通过遥控器手动示教路径后自动循迹行驶。

## 四、核心算法详解

### 4.1 第一层：六轴卡尔曼滤波

**目的**：融合三轴陀螺仪和三轴加速度计，解算出稳定的 **roll（横滚）、pitch（俯仰）、yaw_gyro（航向角）**。

**算法流程**：

```
输入: gyro_x, gyro_y, gyro_z (rad/s)
      acc_x, acc_y, acc_z (m/s?)

Step 1 — 计算欧拉角速率（姿态运动学方程）:
  ω_roll  = gx + sin(roll)·tan(pitch)·gy + cos(roll)·tan(pitch)·gz
  ω_pitch = cos(roll)·gy ? sin(roll)·gz
  ω_yaw   = sin(roll)/cos(pitch)·gy + cos(roll)/cos(pitch)·gz

Step 2 — 时间预测（先验估计）:
  Xk_[i] = Xk[i] + T · Uk[i]          i = 0,1,2

Step 3 — 协方差预测:
  Pk_[i] = Pk[i] + Q[i]                i = 0,1,2

Step 4 — 卡尔曼增益（仅roll/pitch有测量更新）:
  K[0] = Pk_[0] / (Pk_[0] + R[0])     roll通道
  K[1] = Pk_[1] / (Pk_[1] + R[1])     pitch通道
  K[2] = 0                              yaw无加速度测量！

Step 5 — 加速度计测量值提取:
  Zk[0] = atan2(acc_y, acc_z)          roll测量值
  Zk[1] = ?atan2(acc_x, √(acc_y?+acc_z?))  pitch测量值
  Zk[2] = 0                             yaw无测量

Step 6 — 后验估计更新:
  Xk[0] = (1?K[0])·Xk_[0] + K[0]·Zk[0]
  Xk[1] = (1?K[1])·Xk_[1] + K[1]·Zk[1]
  Xk[2] = Xk_[2]                        yaw仅靠陀螺仪积分！

Step 7 — 重力补偿（输出线性加速度）:
  ax_linear = ax ? (?g·sin(pitch))
  ay_linear = ay ? (g·sin(roll)·cos(pitch))
  az_linear = az ? (g·cos(roll)·cos(pitch))

输出: roll, pitch, yaw_gyro, ax/ay/az_linear
```

**关键设计决策**：
- **yaw不参与卡尔曼测量更新**（`K[2]=0`），因为加速度计无法感知水平面内的旋转。这意味着 `yaw_gyro` 是纯陀螺仪积分值，**必然存在随时间累积的漂移**。
- 这正是需要第二层（磁力计校正）的原因。

**默认参数**：
| 参数 | 值 | 含义 |
|------|-----|------|
| Q (系统噪声) | 0.001 | 越小越信任预测模型 |
| R (测量噪声) | 0.1 | 越小越信任加速度计 |
| T (采样周期) | 0.004s | 4ms，跟随实际调用周期 |

### 4.2 第二层：磁力计航向校正

**目的**：利用磁力计提供的绝对航向参考，约束第一层输出的 `yaw_gyro` 的长期漂移。

**算法**：互补滤波（Complementary Filter）

```c
// 1. 从磁力计X/Y轴计算绝对航向
float yaw_mag = atan2f(mag_y, mag_x);

// 2. 计算航向差（归一化到[-π, π]）
float yaw_diff = normalize_angle_rad(yaw_mag - yaw_gyro);

// 3. 互补滤波融合
//    alpha=0.98 表示98%信任陀螺仪（高频响应好），2%信任磁力计（长期稳定）
s_yaw_stable = yaw_gyro + (1.0f ? 0.98f) * yaw_diff;

// 4. 归一化输出
s_yaw_stable = normalize_angle_rad(s_yaw_stable);
```

**为什么选择互补滤波而非扩展卡尔曼**：
- 计算量极小，适合嵌入式实时系统
- 参数直观（一个alpha系数即可调优）
- 对磁力计干扰（金属、电磁噪声）有一定鲁棒性

**磁力计校准（椭球拟合）**：

由于硬铁干扰（永久磁场偏移）和软铁干扰（感应磁场畸变），原始磁力计数据呈椭球分布。系统采用 **最小二乘椭球拟合** 进行校准：

```
采集 N 个样本 (mx_i, my_i, mz_i), i=1..N (N ≥ 100)

求解: min Σ ||M·(p ? b)||? ? r?   其中 p=(mx,my,mz)

步骤:
1. 构建协方差矩阵 M (3×3) 和均值向量 B (3×1)
2. 求解 M?? · B 得到偏置向量 bias (硬铁校正)
3. 计算平均半径 r 得到缩放因子 scale (软铁校正)
4. 校正公式: p_corrected = (p_raw ? bias) × scale
5. 结果保存至 Flash 第63页（带魔数校验和版本号）
```

**实际使用策略**：在最终输出中，系统**始终使用 `yaw_gyro`（陀螺仪积分值）作为位置推算的航向输入**，而非 `yaw_stable`。这是因为磁力计在室内环境中受干扰严重，`yaw_stable` 反而可能引入跳变。`yaw_stable` 主要用于调试监控。

### 4.3 第三层：航位推算（Dead Reckoning）

**目的**：结合航向角和编码器脉冲，递推车辆的 **(x, y)** 平面坐标。

**算法**：差分驱动机器人运动学模型

```c
void ins_position_update(float yaw, int16 tick_left, int16 tick_right)
{
    // 1. 脉冲→位移
    float dist_left  = tick_left  × tick_to_meter_left;   // 左轮位移(m)
    float dist_right = tick_right × tick_to_meter_right;  // 右轮位移(m)

    // 2. 中心点位移（假设无侧滑）
    float dist_center = 0.5f × (dist_left + dist_right);

    // 3. 坐标更新（世界坐标系下的积分）
    s_pos_x += dist_center × cos(yaw);
    s_pos_y += dist_center × sin(yaw);
}
```

**零速修正（ZUPT）**：

当检测到车辆静止时，强制将编码器增量归零，防止微小抖动导致的位置漂移：

```c
static uint8 ins_is_stationary(const INS_Input *input)
{
    return (|v_mps| < 0.05 m/s && |gyro_z| < 0.05 rad/s);
    // 速度和角速度均低于阈值 → 判定为静止
}
```

### 4.4 Pure Pursuit 循迹算法

当系统进入 **循迹模式** 时，采用经典的 **Pure Pursuit（纯追踪）** 算法进行路径跟踪：

```
算法原理:
  在路径上寻找一个"前视目标点"（距离当前位置约 Ld），
  计算连接当前点到目标点的圆弧的曲率，
  将曲率转换为前轮转角。

数学推导:
  1. 目标点转换到车身坐标系:
     x_local = dx·cos(yaw) + dy·sin(yaw)
     y_local = ?dx·sin(yaw) + dy·cos(yaw)

  2. 前视距离: Ld = √(x_local? + y_local?)

  3. 曲率: κ = 2·y_local / Ld?

  4. 转向角: δ = arctan(κ · L)    其中L为轴距

  5. 后处理:
     - 死区: |δ| < 2° → δ = 0 （消除直行抖动）
     - 限幅: |δ| ≤ 20° （物理限制）
     - 低通滤波: output = 0.85·output + 0.15·last （平滑转向）
     - 倒车反转: speed_dir < 0.5 → δ = ?δ
```

**参数配置**：
| 参数 | 值 | 说明 |
|------|-----|------|
| 采样步长 | 0.1m (10cm) | 存点间隔 |
| 前视距离 Ld | 0.4m | Pure Pursuit前视距离 |
| 循迹速度 | 0.35 m/s | 自动循迹时的目标速度 |
| 转向滤波系数 | 0.85 | 一阶低通，越大越跟手 |

**方向切换支持**：每个轨迹点携带 `speed_dir` 标志（1=前进，0=倒车），循迹时自动检测方向变化并切换行驶方向。

## 五、数据流全链路

以一次完整的 **4ms 中断周期** 为例，展示数据从传感器到执行的完整流转：

```
═══ 4ms 周期开始 ═══

① IMU数据采集 (date_handle)
   ├─ imu963ra_get_gyro()     → 原始角速度 (°/s)
   ├─ imu963ra_get_acc()      → 原始加速度 (g)
   ├─ imu963ra_get_mag()      → 原始磁力计 (raw)
   │
   ├─ 零偏扣除: value_raw ? bias
   ├─ 单位转换: gyro→rad/s, acc→m/s? (×9.79)
   └─ 一阶低通滤波 (α=0.8): output = α·new + (1?α)·old
       → 存入 imu660.data_Ripen 结构体

② 编码器更新 (encoder_layer_update)
   ├─ 读取TIM5/TIM6计数值并清零
   ├─ 左轮取反（约定前进为正）
   ├─ tick → 米: delta = tick × 0.00002204
   ├─ 累计里程: odom += delta
   └─ 速度计算: speed = delta / 0.004s

③ INS状态更新 (Ins_update)  ← ★ 核心
   │
   ├─ 第一层: ins_kalman_6axis_update()
   │   输入: gyro_x,y,z + acc_x,y,z
   │   输出: roll, pitch, yaw_gyro
   │
   ├─ 第二层: ins_mag_correct_update()  [如果磁力计有效]
   │   输入: yaw_gyro + mag_x, mag_y
   │   输出: yaw_stable
   │
   └─ 第三层: ins_position_update()
       输入: yaw_gyro + tick_left, tick_right
       输出: x, y → 写入 INS_State

④ 速度PID (wheel_pid_update)
   ├─ 误差: e = target_speed ? current_speed
   ├─ PID: out = Kp·e + Ki·∫e·dt + Kd·de/dt
   └─ 输出PWM → motor_control(LB), motor_control(RB)

⑤ 轨迹处理 (track_proc)
   ├─ [存点模式] 每10cm记录(x, y, yaw, dir)到Flash
   └─ [循迹模式]
       ├─ 寻找前视点 find_lookahead_point()
       ├─ 计算转向角 pure_pursuit_calc_steer()
       ├─ servo_set_angle(90° + steer)
       └─ wheel_pid_set_target_speed(±0.35 m/s)

═══ 4ms 周期结束 ═══
```

## 六、轨迹存储机制

### 6.1 Flash存储规划

```
Flash地址空间:
┌──────────┬──────────────────────────────────────────┐
│ Page 0   │ 元数据页 (Magic + Version + 点数 + 索引)   │
├──────────┼──────────────────────────────────────────┤
│ Page 1~24│ 轨迹数据页 (每页127个点 × 4个float)        │
│          │ 总容量: 24页 × 127点 = 3048个轨迹点        │
│          │ 最大记录距离: 3048 × 0.1m ≈ 305米          │
├──────────┼──────────────────────────────────────────┤
│ Page 63  │ 磁力计校准参数页 (bias + scale)            │
└──────────┴──────────────────────────────────────────┘
```

### 6.2 单个轨迹点格式

每个轨迹点占用 **4个float（16字节）**：

| 偏移 | 字段 | 类型 | 说明 |
|------|------|------|------|
| +0 | x | float | X坐标 (m) |
| +4 | y | float | Y坐标 (m) |
| +8 | yaw | float | 航向角 (rad) |
| +12 | speed_dir | float | 方向标志 (1.0=前进, 0.0=倒车) |

### 6.3 数据完整性保护

- **元数据魔数**：`0x5452434B` ("TRCK") 用于检测有效数据
- **离群点剔除**：GNSS原点计算时剔除偏离均值 >1.5m 的点
- **Flash擦除值检测**：读取时过滤 `0xFFFFFFFF` 和 `0x00000000`
- **浮点有效性检查**：NaN 检测 + 坐标范围限制 (|coord| ≤ 1000m)

### 7.2 典型工作流程

```
步骤1: 进入INS模式
  长按 K4 → 屏幕显示 "INS Control"

步骤2: 磁力计校准（首次使用或环境变化时）
  长按 K2 → 开始校准 → 缓慢旋转车身360°（采集500个点）
  再次长按 K2 → 完成校准 → 参数自动保存至Flash

步骤3: 录制轨迹
  确认处于 "SAVE" 模式 → 短按 K1 → 开始存点

  到达终点后短按 K1（停止）或 K3（结束）

步骤4: 回放循迹
  短按 K2 切换到 "FOLLOW" 模式
  短按 K1 → 小车自动沿录制轨迹行驶
  到达终点自动停止

步骤5: 退出
  长按 K4 → 退出INS模式，回到普通菜单
```

## 八、技术亮点与设计权衡

### 8.1 技术亮点

1. **三层解耦架构**：每层独立可测试，接口清晰。第一层的卡尔曼、第二层的互补滤波、第三层的航位推算各司其职。
2. **椭球拟合磁力计校准**：比简单的极大/极小值法更准确，能同时校正硬铁和软铁干扰。
3. **GNSS原点自动确定**：采集50个样本取均值并剔除离群点，无需手动设置原点坐标。
4. **Pure Pursuit + 方向切换**：支持前进/倒车混合轨迹的完整循迹。
5. **Flash循环写入**：24页循环使用，支持超长轨迹录制（理论最大305米）。
6. **防御性编程**：全面的NULL指针检查、NaN检测、数值限幅、Flash脏数据过滤。

### 8.2 设计权衡与已知局限

| 方面 | 选择 | 原因/影响 |
|------|------|-----------|
| yaw输出选择 | 使用 `yaw_gyro` 而非 `yaw_stable` | 室内磁干扰大，陀螺仪积分短期更可靠；代价是yaw会缓慢漂移 |
| 位置推算方式 | 编码器航位推算（非加速度积分） | 加速度计二次积分发散太快；编码器方式更稳定但依赖轮子不打滑 |
| 卡尔曼维度 | 3状态标量卡尔曼（非矩阵版） | TC377浮点运算能力有限，标量版足够且计算量小 |
| GNSS用途 | 仅作参考，未融入INS闭环 | 室内赛场的GNSS信号质量差，不适合做实时修正 |
| 循迹速度 | 固定0.35m/s | 简化实现，未根据曲率动态调速 |
| 轨迹存储 | 内部Flash | 容量有限（~3000点），外部SD卡方案可扩展 |
